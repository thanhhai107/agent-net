% % As being depicted in the Algorithm \ref{alg:gen-asp}, the input of our framework is a list of stories accompanying with a list of queries which are defined by $\mathcal{SQ}=[(story_i,q_i)]_{1\times n}$, list of the foundation rules $\mathcal{FR} = [fr_j]_{1\times m}$, and the $\epsilon$ used to restrict the number of iterations. The desirable output is a list of answers $\mathcal{A}_{1\times n}$ comprising answers of the given queries.

% % Starting with $\mathcal{A}$ is set to $\O$ because no queries are answered. All input stories and queries are exprienced the semantic parsing process, finally get a list $\mathcal{U}$ containing many elements in the form of $(story,q,semantic)$ where $semantic$ means semantic parses from LLMs. Afterwrads, $\mathcal{M}$ is an ASP solver built from the $\mathcal{FR}$, which can be simply regarded as an ASP system with foundational rules. At the fourth step, $Clingo$ is called by using $\mathcal{M}$ to update $\mathcal{A}$ and $\mathcal{U}$. At this time, $\mathcal{A}$ is a set of answered queries, including both true and false answers, whereas $\mathcal{U}$ is their unanswered counterparts. From this time onwards, $\mathcal{U}$ is utilized to enrich the $\mathcal{FR}$ automatically.

% % While $\mathcal{U}$ is not empty, initially setting $count$ to 0. Next, moving to the first element in $\mathcal{U}$ that is not tried to generate rule before, $ \mathcal{U}[x] $, where $1 \leq x \leq len(\mathcal{U})$. Unless $count > \epsilon$, do the following steps: firstly, a new rule is generated by LLMs through generation stage as mentioned previously; secondly, this new rule is verified whether it is valid or not. If this rule is valid, it will be updated to the $\mathcal{FR}$, and then $\mathcal{M}$ is updated, simultaneously, before breaking from the while loop. By contrast, if this rule conflicts the principle of our conceptual graph, $count$ is added by 1, and the process of generating a new rule is iterated until $count > \epsilon$. 

% % Escaping from this while loop, $count$ is going to be checked. The possible value of $count$ can be ranged from 0 to $\epsilon$. $count$ is equal to $\epsilon$ shows that we have failed to generate a new rule for element $\mathcal{U}[x]$, so $\mathcal{U}[x]$ is marked as tried to avoid unwarranted efforts to offer LLMs to generate a rule for this element again. In a contradictory way, $count$ is different from $\epsilon$ means this generated rule is valid and has been added to the $\mathcal{FR}$. $Clingo$ will be called with the new solver $\mathcal{M}$ to update $\mathcal{A}$ and $\mathcal{U}$. All the above steps will be repeated until there are no story which is not tried, more specifically, when $\mathcal{U}[x]$ is exactly $\mathcal{U}[len(\mathcal{U})]$.

Khi người dùng yêu cầu, phía front-end sẽ gửi yêu cầu tới phía back-end, sau đó back-end sẽ gọi tới module cảnh báo. Ngoài việc đưa ra cảnh báo trực tiếp dựa trên giá trị AQI hiện tại, module này còn có thể đưa ra xu hướng của dữ liệu AQI trong từng khoảng thời gian khác nhau. Đầu vào của module này là một danh sách giá trị AQI theo từng thời điểm (có thể là tương lai hoặc quá khứ). Dữ liệu này sau đó sẽ được đưa qua mô hình trung bình động đơn giản để đánh giá xu hướng của các giá trị AQI này và đưa ra cảnh báo. Xu hướng này có thể là chất lượng không khí đang tốt dần lên, đang xấu dần đi hoặc gần như ổn định.

% Chọn kích thước cửa sổ trượt $k$ tùy vào yêu cầu dự báo xu hướng. Với các dự báo xu hướng trong 6 giờ hoặc 7 ngày (cả trong quá khứ lẫn tới tương lai), lựa chọn $k=4$ là phù hợp cho dự báo xu hướng vì kích thước này là vừa đủ, nếu kích thước quá nhỏ sẽ dễ dẫn đến việc mô hình nhạy cảm với biến động nhỏ hoặc theo giai đoạn nhỏ. Nếu kích thước cửa sổ trượt quá lớn thì sẽ không mang lại nhiều ý nghĩa trong việc nhận xét xu hướng. Đối với các cảnh báo theo ngày, kích thước cửa sổ trượt được chọn là $k=8$. 

% Ví dụ, giả sử bây giờ là 16 giờ $x$ phút ($0 < x \leq 59$) và bạn có dữ liệu trung bình AQI theo từng giờ trong vòng 1 ngày trở lại đây, bạn có thể đưa qua mô hình trung bình động để dự đoán xu hướng. Dự báo đầu ra cho thấy, giá trị AQI có xu hướng tăng lên trong vòng 24 giờ tính đến hiện tại, do đó chất lượng không khí được cảnh báo có xu hướng xấu đi cũng trong khoảng thời gian đó.
